66. Plus One

Given a non-negative integer represented as a non-empty array of digits, plus one to the integer.
You may assume the integer do not contain any leading zero, except the number 0 itself.
The digits are stored such that the most significant digit is at the head of the list.

Solution 1

First, check from the last digit to see if it is 9 or not. If it’s not, break out of the loop. If it is 9, then we need to carry it to 10 which means set this digit to 0. Loop over to the next digit and check again until it reaches the first digit.

If i >= 0 after the loop, this means ith digit is not 9 and we break out of the loop. Thus, add 1 to ith digit.
If i < 0 after the loop, this means the first digit of the array is 9 and in order to carry it to 10, we need to add digit 1 to the beginning of the array.

Time Complexity：O(n)

Solution 2

Set a carry variable to 0 and loop from the end of the array. First, we add 1 to the last digit. Then, add carry(0/1) to every digit when looping over the array. If ith digit is 10 then set digit to 0 and carry to 1. If the carry is 1 after the loop, we need to add the carry to the begining of the array.

Time Complexity：O(n).

53. Maximum Subarray

Find the contiguous subarray within an array (containing at least one number) which has the largest sum.

For example, given the array [-2,1,-3,4,-1,2,1,-5,4],
the contiguous subarray [4,-1,2,1] has the largest sum = 6.

More practice:
If you have figured out the O(n) solution, try coding another solution using the divide and conquer approach, which is more subtle.

Solution 1

Dynamic programming(DP)/Kadane’s algorithm
Maximum problem can be seen as optimization problem, thus we can use DP to solve this. Defined on Wiki, DP is a method for solving complex problems by breaking them down into simpler subproblems. It is applicable to problems exhibiting the properties of overlapping subproblems and optimal substructure. This question is only exhibit optimal substructure.

Reference to the discussion here. The subproblem is to calculate sum(0,i) which can be either adding sum(0,i-1) to nums[i], or not. If sum(0,i-1) is negative, then add it to nums[i] only makes the sum smaller. Thus, we add only if it’s not negative.

Time Complexity：O(n)
Space Complexity：O(1)

However, there is a name for this method called Kadane’s algorithm. We can simply consider that max_ending_here[i+1] = max(nums[i], nums[i] + max_ending_here[i])

Time Complexity：O(n)
Space Complexity：O(1)

Solution 2

Divide and Conquer algorithm is another way to solve this problem. This method uses recursion to break a problem into sub-problems until they can be solved directly. The solutions are then put together to solve the original problem.

Reference to discussion here and book Introduction to Algorithms, Third Edition, Chapter 4.1.
First, we equally divide the array into two subarrays by finding the midpoint, then the maxSubarray A[i,j] must lie in following three places:

  • entirely in the left subarray A[left,middle], so that left<=i<=j<=middle.
  • entirely in the right subarray A[middle+1,right], so that middle+1<=i<=j<=right.
  • crossing the midpoint, so that left<=i<=middle<=j<=right.

If crossing the midpoint, then the result will be adding the maximum suffix subarray of the left subarray and the maximum prefix subarray of the right subarray.
If not crossing the midpoint, then we can just apply the same algorithm to the left and the right part respectively to find the max subarray.
Finally, return the maximum of those three situation.

Time Complexity：O(nlogn).

26. Remove Duplicates from Sorted Array

Given a sorted array, remove the duplicates in-place such that each element appear only once and return the new length.
Do not allocate extra space for another array, you must do this by modifying the input array in-place with O(1) extra memory.

Example:

Solution 1

Delete duplicate elements from Array using Array.Splice().

Solution 2

For a sorted array, we can use two pointers i, j to find the duplicate.
If nums[i] == nums[j], we increment j to skip the duplicate.
If nums[i] !== nums[j], we know the duplicate run has ended so we copy the new value nums[j] to nums[i+1].
i is then incremented and we repeat until j reached the end of the array.

Time Complexity：O(n)
Space Complexity：O(1)

1. Two Sum

Given an array of integers, return indices of the two numbers such that they add up to a specific target.
You may assume that each input would have exactly one solution, and you may not use the same element twice.
Example:

Solution 1

Simplest way is to iterate the array twice and sum up to find the target.

Time Complexity：O(n^2)
Space Complexity：O(1)

Solution 2

Use indexOf() method to search the array.

Time Complexity：O(n^2)

Solution 3

This one is the optimal solution.
Create a hash table and store the array value(key) and corresponding index(value). Only iterate the array once. For each element, check if there is a match in the hash table. If not, add this element to the hash table.

Time Complexity：O(n)
Space Complexity：O(n)
